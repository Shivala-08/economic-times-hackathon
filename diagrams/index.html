Overview: The system consists of five main layers: Streamlit UI (front-end), FastAPI Backend (API layer), Query Engine Pipeline (retrieval + generation), Storage (ChromaDB vectors + NetworkX graph + BM25 sparse), and LLM Providers (NVIDIA NIM primary, Ollama fallback). The semantic cache sits between retrieval and LLM to skip redundant calls.

🔍 Query Pipeline

Retrieval Re-ranking
graph TD Q["❓ User Query"] --> Embed["📐 Embed Query\n384d vector"] Embed --> ChromaDB["🟦 ChromaDB\nTop-16 Candidates\n(Diverse Search)"] Embed --> BM25["📝 BM25\nSparse Scores"] ChromaDB --> Hybrid["🔀 Hybrid Fusion\nα = 0.4 × BM25\n+ 0.6 × Vector\nCosine Similarity"] BM25 --> Hybrid Hybrid --> CE["🎯 Cross-Encoder\nms-marco-MiniLM-L-6-v2\n(Re-rank Top-16 → Top-3)"] CE --> KW["🏷️ Keyword Boost\n+3 pts per match\nMax +12 pts"] KW --> Top3["📊 Top-3 Chunks\nFinal Candidates"] Q --> Extract["🏷️ Entity Extraction\nspaCy + Regex"] Extract --> Graph["🕸️ Graph Traversal\n1-hop Neighbors\n(Cap: 20 entities)"] Top3 --> Context["📝 Context Assembly\nDocument Context\n+ Graph Context\n+ Confidence Guide"] Graph --> Context Context --> LLM["🤖 LLM Generation\nNVIDIA NIM\nnemotron-3-ultra-550b"] LLM --> Answer["✅ Answer\nSources + Confidence\n+ Key Points"] style Q fill:#6366f1,color:#fff style ChromaDB fill:#3b82f6,color:#fff style Hybrid fill:#10b981,color:#fff style CE fill:#f59e0b,color:#000 style Graph fill:#ec4899,color:#fff style LLM fill:#8b5cf6,color:#fff style Answer fill:#10b981,color:#fff
Query Pipeline Detail: The query is embedded and fed into both the vector store and BM25 index. Results are fused (α=0.4 BM25, 0.6 vector), re-ranked by cross-encoder, boosted by keyword matches, and narrowed to top-3 chunks. Simultaneously, entities are extracted for graph traversal. Both contexts are assembled into the final prompt.

Optimization Journey: Three iterative tiers transformed the system from 77.8% accuracy / 10.3s latency to 100% accuracy / 771ms latency. Tier 1 fixed correctness (graph context, chunking, scoring). Tier 2 optimized latency (re-ranking, caching, complexity classification). Tier 3 added streaming UX (SSE, error recovery, real-time rendering).

Streaming Sequence: The full request lifecycle from UI to LLM and back. Context retrieval happens synchronously, then tokens stream via SSE. The try/finally block ensures metadata and done events always fire, even on mid-stream errors. First token latency is under 100ms.
